# daily-ai-news Wiki
daily-ai-news 是一款基于**豆包大模型（火山引擎）**的自动化每日AI资讯生成与推送工具，通过 GitHub Actions 实现无服务器化定时运行，具备**终身去重**能力，支持多收件人邮件推送。

## 核心功能
### 1. 豆包大模型智能生成资讯
- 基于火山引擎方舟平台的 `Doubao-Seed-2.0-pro` 模型生成高质量AI资讯
- 固定分6大模块：大模型与技术突破、算力与硬件、医疗与科学AI、资本与产业、安全与监管、端侧与应用
- 自动生成核心资讯总览目录（单条20字内），每条资讯包含标题、核心内容、核心影响、可验证信源
- 关键信息自动加粗，段落清晰易读

### 2. 终身去重机制
- 内置 `news_blacklist.json` 去重库，自动记录所有已生成资讯的指纹
- 每次生成前自动加载历史去重库，严格禁止重复内容
- 生成后自动追加新指纹并提交回仓库，实现终身去重

### 3. 多收件人邮件推送
- 支持单个/多个收件人，英文逗号分隔即可
- 邮件采用专业HTML样式，适配手机/电脑端显示
- 完全兼容QQ邮箱格式，不会触发反垃圾邮件规则

### 4. GitHub Actions 自动化定时运行
- 默认北京时间每天9:00自动运行（可自定义）
- 无需本地服务器，完全免费使用GitHub Actions额度
- 支持手动触发测试，运行日志清晰可查

## 关键 Secret 配置（必看）
所有 Secret 需在 GitHub 仓库的 `Settings → Secrets and variables → Actions → New repository secret` 中配置（名称严格区分大小写），以下是完整配置说明：

### 豆包API配置（必配）
#### 1. DOUBAO_API_KEY
- **作用**：调用火山引擎豆包大模型API的身份凭证
- **获取方式**：
  1. 登录[火山引擎官网](https://www.volcengine.com/)，完成实名认证
  2. 进入[火山方舟大模型服务平台](https://www.volcengine.com/ark)
  3. 左侧菜单【系统管理】→【API密钥管理】→【创建密钥】
  4. 复制生成的完整API Key（仅显示一次，需妥善保存）
- **配置要求**：直接粘贴生成的API Key，无需额外处理

#### 2. DOUBAO_ENDPOINT_ID
- **作用**：指定调用的豆包模型端点ID
- **获取方式**：
  1. 进入火山方舟控制台，左侧菜单【模型推理】→【在线端点】
  2. 点击【创建端点】，基础模型选择 `Doubao-Seed-2.0-pro`
  3. 计费模式选「按量计费」，模型算力计费开关保持关闭
  4. 创建完成后，复制 `ep-` 开头的端点ID
- **配置要求**：粘贴完整的 `ep-` 开头的端点ID

### 邮件推送配置（必配）
#### 3. SENDER_EMAIL
- **作用**：发件人邮箱地址
- **配置要求**：粘贴完整邮箱地址，例如 `123456@qq.com`

#### 4. EMAIL_AUTH_CODE
- **作用**：发件邮箱的SMTP授权码（非登录密码）
- **获取方式**（以QQ邮箱为例）：
  1. 登录[QQ邮箱官网](https://mail.qq.com/)
  2. 顶部【设置】→【账户】→ 找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务」
  3. 开启「POP3/SMTP服务」，按提示完成密保验证
  4. 复制生成的16位纯英文授权码
- **配置要求**：粘贴生成的授权码，不要带空格

#### 5. RECEIVER_EMAIL
- **作用**：收件人邮箱地址，支持多个收件人
- **配置要求**：
  - 单个收件人：直接粘贴邮箱地址，例如 `aaa@qq.com`
  - 多个收件人：用英文逗号分隔，例如 `aaa@qq.com,bbb@163.com,ccc@gmail.com`

#### 6. SMTP_SERVER
- **作用**：发件邮箱的SMTP服务器地址
- **配置要求**：
  - QQ邮箱：`smtp.qq.com`
  - 163邮箱：`smtp.163.com`
  - 腾讯企业邮箱：`smtp.exmail.qq.com`
  - 其他邮箱：查询对应邮箱官网的SMTP服务器地址

## 使用步骤
### 1. 准备工作
1. 注册GitHub账号，Fork本仓库（或直接使用本仓库）
2. 注册火山引擎账号，完成实名认证，开通豆包大模型服务
3. 准备一个发件邮箱（推荐QQ邮箱），开启SMTP服务并获取授权码

### 2. 配置Secret
按上文「关键Secret配置」说明，在仓库中配置全部6个Secret。

### 3. 初始化去重库（可选）
仓库已包含初始的 `news_blacklist.json` 去重库，如需自定义，可直接编辑该文件，每行一个去重指纹。

### 4. 触发运行
- **自动运行**：GitHub Actions 会按默认配置（北京时间每天9:00）自动运行
- **手动运行**：
  1. 进入仓库「Actions」页面
  2. 左侧选择「每日AI资讯自动推送（含终身去重）」
  3. 点击「Run workflow」→ 再次点击「Run workflow」手动触发

### 5. 验证成功
- 查看Actions日志，依次看到：
  - ✅ 开始生成xxxx AI资讯，已加载xxx条历史去重指纹
  - ✅ 资讯生成成功，内容长度：xxxx
  - ✅ 提取到30条新去重指纹
  - ✅ 邮件发送成功
  - ✅ 永久去重库已更新
- 打开收件邮箱，确认收到格式清晰的AI日报

## 自定义配置
### 修改定时运行时间
编辑 `.github/workflows/daily-news.yml` 文件中的 `cron` 表达式（GitHub Actions使用UTC时区，北京时间=UTC+8）：
```yaml
on:
  schedule:
    - cron: '0 1 * * *'  # UTC 1:00 = 北京时间 9:00
```
例如：
- 北京时间每天早8点：`cron: '0 0 * * *'`
- 北京时间每天中午12点：`cron: '0 4 * * *'`

### 修改资讯生成规则
编辑 `news_generator.py` 文件中的 `build_prompt` 函数，可自定义：
- 资讯模块数量和顺序
- 单条资讯的格式
- 去重规则
- 生成的Token数量和温度

## 常见问题
### Q1: 如何查看运行日志？
1. 进入仓库「Actions」页面
2. 点击对应的运行记录（绿色对勾=成功，红色叉号=失败）
3. 点击左侧「build」，查看完整的实时日志
